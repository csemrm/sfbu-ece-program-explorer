# Development Workflow

## Startup

Whenever Claude starts:

1. Read
   - CLAUDE.md
   - PROJECT_ROADMAP.md
   - TASKS.md
   - CHANGELOG.md

2. Determine

- Current milestone
- Current sprint
- Next unfinished task
- Dependencies

3. Summarize project status.

4. Recommend ONE implementation task.

Wait for approval.

---

## During Development

Before modifying code:

- Explain the plan.
- List files that will change.
- Explain why.

Only modify files related to the current task.

Do not refactor unrelated code.

---

## After Development

After completing work:

- Update TASKS.md
- Update CHANGELOG.md
- Mark completed tasks
- Suggest next task

Never skip documentation.
